## 🛑 部署与排雷操作指南 
*(Box / Surfing / FlClash / ShellCrash 用户必读)*

> [!WARNING]
> **⚠️ 致命冲突警告**
> 本配置文件兼容纯血 Mihomo 内核的所有客户端，可以与 RDP / Tailscale / Rustdesk / VPN 等共存。
> **但在同一台设备上，绝对不要同时运行两套流量接管机制（例如同时开启底层模块和 FlClash）！**

### 📱 Box / Surfing 手机端操作步骤
如果您决定使用 **Box for Root / Surfing 模块** (无感静默运行)：
1. 确保面板设置中**关闭**了【自动复写】。
2. 找到“地区自动代理组”模板：
   - ✅ **启用** ⚠️ `Smart` 智能策略代码。
   - ❌ **注释或删除** ⚠️ `URL-Test` 部分代码。
3. 将本文件直接替换掉 Box 目录下的 `config.yaml`，然后重启模块。
4. 🚫 **切记**：不要在手机上再打开 FlClash 去接管 VPN！

### 📱 FlClash 手机端操作步骤
如果您决定使用图形界面的 **FlClash 客户端**：
1. 必须前往系统设置或 Magisk/KernelSU，**关闭或卸载** Box for Root、Surfing 等底层透明代理模块。
2. **重启手机**，确保底层路由表完全清空。
3. 找到“地区自动代理组”模板：
   - ✅ **启用** ⚠️ `URL-Test` 部分代码。
   - ❌ **注释或删除** ⚠️ `Smart` 智能策略代码。
4. 打开 FlClash，导入本文件，点击启动即可 (FlClash 会自动拉起 VPN 模式接管流量)。

### 💻 FlClash 电脑端操作步骤
1. 找到“地区自动代理组”模板：
   - ✅ **启用** ⚠️ `URL-Test` 部分代码。
   - ❌ **注释或删除** ⚠️ `Smart` 智能策略代码。
2. 将本文件导入电脑端 FlClash。
3. 开启 **"TUN 模式"** 或 **"系统代理"** 即可直接运行。

### 🌐 ShellCrash 路由端操作步骤
1. 在配置文件中找到 `tproxy-port: 7895`，将其修改为 `tproxy-port: 9898`。
2. 导入配置，并按 ShellCrash 的提示进行操作。
   > *(注：目前作者尚未在路由端进行实测，请谨慎操作)*
